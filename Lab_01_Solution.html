Cogitive Process Chain

The Cognitive Process chain grabbed my attention from the last class. It shows how human mental state goes through series of Process to finally make a desicion. And the class activity in which we checked all the necessary steps for problem solving was interesting and it boosted our cognitive thinking capability.

Automatic Syringe

The Automatic Syringe example demonstrate how important is designing and as humans can make mistake so the effect of one accidental key slip can lead to out of factor dose in the older numeric keypad. Therefore it says that every user should be involved in designing process of any equipments or machinery.

After consulting users, it solves the problem by making individual increment/decrement buttons for each digit. This reduces chances of accidental keypress as we can set each digit separately. This shows that how a simple user interface is clear, effective and easy to use.

Reference: HCI Book by Alan Dix. Topic: Design Focus, page number 7-8.

Who Paid Problem

The Who Paid Problem highlights about a social coordination system where design choices have direct emotional impacts on friendships. It solves the problem that evey person in friends group feels shy and cheap to discuss about the bill when they go for dinner or buy something from stores or spend on trip. It does not feel good to ask about cost per person and distribute money. And it feels frustating when friends forget to pay for their bill.

So this shows how an interactively designed app can solve this issue that ensures financial accuracy and can manage splitting bill among friends within short time with minimizing the social awkwardness for asking friends for money.

Part C

  • Explain the difference between a semantic element (e.g., nav) and a generic element (e.g., div).

    • Semantic element: Has meaning and purpose built into HTML. Example: <nav> (navigation), <header>, <article>. It helps browsers, search engines, and screen readers understand the content.
    Generic element: Has no inherent meaning, used only for structure or styling. Example: <div>. You have to rely on classes, IDs, or CSS for meaning.

  • Why does KLM not model high-level planning and interpretation?
    • KLM (Keystroke-Level Model) focuses on low-level, observable actions: keystrokes, clicks, mouse movements, mental operators ([M]), etc.
    • It assumes the user is an expert and knows what to do, so decision-making, planning, and interpreting the task are ignored.
    • Purpose: to predict task execution time for skilled, error-free users.
